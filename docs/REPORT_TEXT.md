## Название проекта

Веб-система управления сетью сайтов с SEO-мониторингом и планированием перелинковки (SiteNet Manager)

## Описание (3–5 предложений)

SiteNet Manager — веб-приложение для учёта и управления группами связанных веб-сайтов в рамках SEO-кампаний. Система позволяет создавать сети проектов, добавлять сайты, выполнять автоматический SEO-анализ страниц, мониторить доступность ресурсов (HTTP-коды 200/301/404) и планировать перелинковку между сайтами сети. Дополнительно реализованы модули семантического ядра (облако ключевых слов с показателями конкуренции и сложности продвижения) и легальной работы с поведенческими факторами (улучшение UX и контента без накрутки). Реализованы REST API, реляционная база данных PostgreSQL, авторизация пользователей и экспорт sitemap.xml и CSV-матрицы ссылок.

## Стек

- Frontend: React 19, Vite, TypeScript, Tailwind CSS
- Backend: Node.js, Express, TypeScript, Prisma ORM
- БД: PostgreSQL
- Документация API: Swagger UI
- Деплой: Vercel (frontend) + Render (backend + PostgreSQL)

## Use Case 1 — Создание SEO-сети

1. Пользователь входит в систему
2. Переходит в раздел «Сети»
3. Заполняет название и описание
4. Нажимает «Создать сеть»
5. Сеть появляется в списке и на дашборде

## Use Case 2 — Добавление сайта

1. Пользователь выбирает сеть
2. Вводит домен, название, нишу, теги
3. Сохраняет сайт
4. Сайт отображается в списке сети

## Use Case 3 — SEO-анализ

1. Открывает карточку сайта
2. Нажимает «SEO-анализ»
3. Система парсит URL и сохраняет title, description, h1, robots.txt
4. Отображается SEO score и история проверок

## Use Case 4 — Матрица перелинковки

1. Открывает сеть → вкладка «Матрица ссылок»
2. Выбирает сайт-источник и сайт-назначение
3. Указывает anchor text и тип ссылки
4. Матрица обновляется, связь сохраняется в БД
5. Доступен экспорт CSV

## Дополнительные модули (для защиты)

### Мониторинг HTTP-статусов

На главной панели отображаются коды ответа всех сайтов сети (200 — доступен, 301 — редирект, 404/5xx — ошибка). Кнопка «Проверить все» запускает массовую проверку uptime.

### Семантическое ядро

Для каждого сайта — облако ключевых слов с частотностью Google/Яндекс, процентом конкуренции в SERP и сложностью продвижения по каждому ключу.

### Поведенческие факторы (white-hat)

Модуль оценивает метрики вовлечённости (отказы, время на сайте, глубина просмотра) и формирует план легальных улучшений UX и контента. Накрутка кликов и ботов не используется.

## Рефлексия (для отчёта)

Проект позволил закрепить навыки проектирования БД, разработки fullstack-приложения и развёртывания веб-сервисов.
